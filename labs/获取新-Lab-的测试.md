
# Git 同步新 lab 教程（针对初学者）

## 前提

1. 你已经 **clone** 了老师提供的仓库：

   ```bash
   git clone <老师仓库URL>
   cd <仓库名>
   ```
2. 仓库结构：

   ```
   ├── vsrc/   # 你写代码的地方
   └── 其他  # 老师更新的测试
   ```
3. 老师每次发布新 lab，只会 **更新 其他 目录**。

---

## 一、提交自己当前的代码（保存进度）

在做任何同步操作前，先把自己写的代码 commit 保存：

```bash
git add .
git commit -m "完成 Lab X 的作业"
```

> 💡 注意：
>
> * `git add .` 会把所有修改都加进暂存区
> * commit 信息可以写清楚自己的进度

---

## 二、获取老师的新更新

1. 获取远程仓库最新内容（不会自动合并）：

   ```bash
   git fetch origin
   ```

2. 查看老师最新的分支：

   ```bash
   git log origin/main --oneline
   ```

   > 假设老师的主分支叫 `main`。

---

## 三、方法 1：使用 Merge（推荐初学者）

1. **切换到自己当前分支**（通常是 `main` 或 `master`）：

   ```bash
   git checkout main
   ```

2. **合并老师的最新更新**：

   ```bash
   git merge origin/main
   ```

3. 处理冲突（如果有的话）：

> 理论上不应该出现，老师不会改动 `vsrc` 目录，但如果老师更新了其他目录，而你也修改了同一文件，就会有冲突。

   * Git 会提示哪个文件冲突
   * 打开冲突文件，修改冲突部分，保留老师的更新和自己的修改。
   * 冲突解决后：

     ```bash
     git add <冲突文件>
     git commit
     ```

1. 完成后，你的 `vsrc` 代码和老师的新测试文件都在本地了。

> 👍 优点：安全，容易理解；不会改变你的提交历史。

---

## 四、方法 2：使用 Rebase（进阶一点）

Rebase 会把你本地的提交“放到”老师更新之后，保持提交历史更整洁。

1. **切换到自己的分支**：

   ```bash
   git checkout main
   ```

2. **拉取并 rebase 老师的更新**：

   ```bash
   git fetch origin
   git rebase origin/main
   ```

3. 如果出现冲突：

   * Git 会停下来提示冲突文件。
   * 修改冲突文件，保留自己的 `vsrc` 改动。
   * 然后：

     ```bash
     git add <冲突文件>
     git rebase --continue
     ```
   * 直到 rebase 完成。

> ⚠️ 注意：
>
> * Rebase 会改变你的提交历史，建议初学者在本地操作，推送前先确认没问题。
> * 如果遇到复杂冲突，不熟练的同学可以先用 merge。

---

## 总结

| 方法     | 特点                   | 建议对象       |
| ------ | -------------------- | ---------- |
| Merge  | 安全，不改提交历史，容易理解       | 初学者        |
| Rebase | 提交历史整洁，把自己的改动放到最新更新后 | 熟悉 Git 的学生 |

> 💡 小技巧：
>
> * 每次更新前，先 commit 自己的代码
> * 遇到冲突不要慌，认真对比保留老师更新和自己的改动
> * 只更新老师的内容，`vsrc` 保留自己写的代码
