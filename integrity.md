# 关于学术诚信与抄袭

本课程实验须由选课学生独立完成。以下就实验设计特点、抄袭与雷同的审查方式及处理办法作正式说明，请务必在提交前知悉并遵守。

## 实验设计特点

本学期实验为**按步骤、自零起搭建**的 RISC-V CPU 实现流程，各 Lab 依次依赖前一阶段的代码与设计，整体具有连贯性。因此：

- 无法通过「在不同来源分别获取不同 Lab 的代码并拼凑」等方式规避审查，整体实现逻辑与代码结构将体现是否为本人在本课程框架下独立完成。
- 批改与抽查时，除验证测试通过情况外，将重点审查**实现思路、模块划分、信号命名与代码结构**是否具有一致性及合理性。

## 审查方式

为保障公平，课程助教将采取以下措施：

- **收集与本课程相关的历年开源实验仓库**（包括但不限于 GitHub、GitLab 等平台），通过爬虫等手段建立可检索的代码库。
- **在人工判定的基础上，采用 agent 等工具进行批量辅助审查**，对实现逻辑、控制流、数据流及代码结构进行比对，而非仅依据文本相似度。
- 重点核查**是否存在多份作业在实现逻辑上高度一致**的情形，例如仅存在变量名、注释或局部语法差异，而整体设计、模块划分与关键逻辑基本相同。

审查将结合自动化工具与人工分析进行，以形成可依据的判定结论。

## 认定与处理

经认定存在抄袭或大面积雷同的，包括但不限于：与历年开源代码或与本届其他同学提交在**实现逻辑上大部分相同**，或仅进行变量名替换、格式调整等简单改写的，作如下处理：

- 取消该次实验成绩；
- 情节严重者将影响课程总评，并按规定上报学校有关部门。

允许在思路层面进行讨论与交流，但**代码及实现逻辑须独立完成**，不得照抄或照搬他人或开源实现。

## 其他说明

遇有疑问可查阅课程资料、RISC-V 手册，或通过课程 Issues、群聊及线下答疑向助教咨询。若在实现过程中参考过外部博客、开源代码等，须在实验报告中**如实注明参考来源及本人修改与贡献**；未如实注明或冒充原创的，经查实后按抄袭处理。

对上述条款有疑问的，可在课程群或答疑时间向助教提出。
