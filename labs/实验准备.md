# 实验准备

## 实验环境搭建与介绍

本学期的课程，我们需要在自己的电脑上安装：

- Linux 环境（推荐Ubuntu 20.04或更高版本）
- Verilator
- GTKWave
- Vivado 2018.3（其它版本可能会有不兼容）

### Linux 环境安装

有三种方案：

- Windows+WSL
- Linux 虚拟机（VMware）
- Linux 原生系统

如果你没安装过后两种，那么推荐第一种方案，以下给出安装步骤：

1. 在 Windows 系统上打开 Microsoft Store
2. 搜索 Ubuntu 并安装，安装后请在开始菜单找到 Ubuntu 应用并打开，初次打开可能会进行一段时间的安装，并要求配置用户名和密码，请按照提示操作。可参考[WSL 安装教程](https://learn.microsoft.com/zh-cn/windows/wsl/setup/environment#set-up-your-linux-username-and-password)的设置 Linux 用户名和密码部分。
3. （可选）配置 apt 源（修改 /etc/apt/sources.list ，推荐中科大源或清华源，具体步骤请参考[USTC Mirror Help](https://mirrors.ustc.edu.cn/help/ubuntu.html)或[清华大学开源软件镜像站](https://mirrors.tuna.tsinghua.edu.cn/help/ubuntu/)）
4. 更新 apt 包数据（运行命令：`sudo apt update`）

> 在 Linux 使用 `sudo` 时（即命令最前面以 `sudo` 开头），可能会要求你输入密码。输入密码时，终端不会显示任何字符（包括星号），请直接输入密码并按回车键即可。

> 至于我们为什么要用 Linux，请看下一部分：Verilator。

### 仿真器 Verilator

Verilator 是一个开源的 Verilog HDL 模拟器，能够以电路级别的方式模拟执行 SystemVerilog 代码。

我们采用 Verilator 对 CPU 进行模拟，然后用 difftest（已内置在实验仓库，无需特意安装）进行指令级别层面的对比。对于你来说，直接在仓库目录下利用 makefile 执行 make 相关命令即可。

虽然verilator的仿真速度一般而言更快，但是功能还不完善，因此对我们编写代码有一定的限制，请参考对应部分的 Wiki。

需要 verilator >= 4.200 版本。Verilator 没有 Windows 版本的，需要在 Linux 环境中运行。

无论你使用什么版本的 Linux，我们都推荐从源码进行编译。

#### 安装依赖

请依次输入以下命令，若需要确认（即要求输入[y/n]），请输入 y 并回车，若需要输入密码，请输入密码：
```
sudo apt update
sudo apt install git help2man perl python3 make autoconf g++ flex bison ccache libgoogle-perftools-dev numactl perl-doc
sudo apt install libsdl2-dev
sudo apt install libfl2  # 如果报错，可以忽略
sudo apt install libfl-dev  # 如果报错，可以忽略
sudo apt install zlib1g zlib1g-dev  # 如果报错，可以忽略
```

```
cd ~
git clone https://github.com/verilator/verilator.git #卡了可以换成https://gitlab.com/fudan-systa/verilator.git
cd verilator
git checkout v5.032
autoconf
./configure
make -j12  #如果这一步出现问题，改成make -j4或者直接make
sudo apt remove verilator #卸载已有的版本
sudo make uninstall
sudo make install
verilator --version  #查看verilator的版本
```

如果报某个头文件 not found，请参考 [https://verilator.org/guide/latest/install.html#git-quick-install](https://verilator.org/guide/latest/install.html#git-quick-install)

### 波形图软件 GTKWave

我们将使用 GTKWave 软件查看仿真波形图。

#### 在 Linux 环境中安装 GTKWave

如果你使用以下之一：

- Linux 虚拟机（带有 GUI）
- Linux 原生系统（带有 GUI）
- 较新版本的 WSL（包含 WSLg，这要求你的电脑运行 Windows 11 或 Windows 10 的较新版本）

只需：

```
sudo apt install gtkwave
```

运行 GTKwave：

```
gtkwave
```

#### 在 Windows 环境中安装 GTKWave

如果使用不带 GUI 的 Linux 环境（旧版本的 WSL，或者你的 WSLg 不知为何无法显示窗口），则需要安装 GTKwave 的 Windows 版本，点击[此处](https://gitlab.com/fudan-systa/arch-2025/-/wikis/uploads/gtkwave.zip)下载压缩文件，使用时，双击 fst 波形图文件，设置 gtkwave/bin/gtkwave.exe 为默认打开程序即可。

## 配置 Visual Studio Code

我们推荐使用 Visual Studio Code 进行代码编写和调试。你可以使用自己喜欢的编辑器并自行配置代码高亮等功能，此处我们仅提供 Visual Studio Code 的配置方法。

### 安装 Visual Studio Code

如果你没有使用过 Visual Studio Code，可以在 https://code.visualstudio.com/ 下载并安装。

具体操作，请参阅 [Microsoft 教程](https://learn.microsoft.com/zh-cn/training/modules/install-configure-visual-studio-code/2-exercise-download-install-visual-studio-code) 的 2~4 单元。

如果你已经安装了 Visual Studio Code，并发现其是英文，请在左侧找到 Extensions（扩展）标签，搜索 Chinese Language Pack for Visual Studio Code，安装后重启 Visual Studio Code 即可。

### 连接到 Linux 环境

如果你使用带有 GUI 的 Linux 虚拟机或 Linux 原生系统，并在 Linux 本机使用 Visual Studio Code，则不需要进行连接，直接在 Linux 环境中打开 Visual Studio Code 即可。

对于 Windows + WSL 的用户，或希望在 Windows 上连接到 Linux 虚拟机的用户，可以使用 Visual Studio Code 的远程连接功能。（如果使用虚拟机，你需要知道如何通过 SSH 连接到虚拟机，关于虚拟机的网络配置，请自行查找相关资料）

你需要安装相关插件，请参考 [Microsoft 教程](https://learn.microsoft.com/zh-cn/windows/wsl/tutorials/wsl-vscode#install-vs-code-and-the-wsl-extension)。

安装好后，在 Visual Studio Code 的左下角有一个连接按钮。

![image](Environment/remote_button.png)

点击后，选择连接到 WSL（如果你使用的是 WSL）或连接到主机（如果你使用的是 Linux 虚拟机）。

## 克隆实验代码仓库

我们采用 git 工具 对所有的 lab 进行管理并发布（推荐git graph插件）。以下是克隆/更新代码仓库的步骤：

请先随便创建一个你需要将代码放置的位置（也是你日后进行代码编写的位置），例如：`~/arch_2026`

你需要运行 `mkdir ~/arch_2026` 来创建这个文件夹（如果已经存在了就不需要了），然后进入这个文件夹：`cd ~/arch_2026`。

然后运行以下命令：

```
git clone https://github.com/26-Arch/26-Arch.git # 克隆代码仓库
cd 26-Arch # 进入代码仓库
git submodule update --init --recursive #更新子仓库，即difftest
```

每次发布新 lab 后，你需要拉取助教上传的最新测试并合并到你写的代码中：

请在操作前，commit 你本地的改动（如果有的话），以免丢失。如果你使用 visual studio code，请确保“更改”标签页中没有未 commit 的改动。否则，你需要先 commit 这些改动（可以先随便写个 commit message），然后再进行以下操作：

```
git fetch --all  # git fetch origin main 只拉取需要的main分支
git merge origin/main  # merge 前记得 commit 自己本地的改动，自行解决冲突
git submodule update #更新子仓库，即difftest
```
